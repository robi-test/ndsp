This section explains how the permission request process works, what information you will need to provide, and what happens after you submit a request.

Note: It is important that you agree the role you need with your team before starting the request.

You are new to NDSP and currently have no permissions

If you sign into NDSP and do not yet have any permissions, you will land on a simplified home page.

Select 'Start a permission request' to begin.

What you need to provide

The first screen explains what you will be asked to provide:

  • The organisation (and its ODS code) you will be submitting data for
  • The role you are requesting
  • If required, details for your organisation's Information Governance (IG) Lead

This page includes a link to the ODS portal, where you can check your organisation's ODS code and confirm the details of its registered IG Lead.

Click 'Continue' to proceed.

Choose a collection

Next, you select the data collection you need permissions for.

The screen shows only collections available through NDSP at the time.

Once you have selected the collection, click 'Continue'.

Select an organisation

You must now choose the organisation you want permissions for:

  • Start typing your organisation's name into the search box. Results will appear once you enter at least three characters.
  • Select your organisation from the list. The list only includes organisations authorised to submit data for the collection you selected.
  • Check that the correct ODS code appears automatically beneath the organisation name. Make sure it matches the organisation you intend to request access for.
  • Once you are confident that the organisation and ODS code are correct, select 'Continue' to move to next step.

If the organisation you expect is not listed, you may need to talk to your line manager or contact the national service desk.

Select a role

You now choose the role you are applying for:

At this stage:

  • Review the list of available roles shown as radio buttons. The roles displayed depend on the organisation and collection you selected.
  • Select the role you need by choosing the appropriate radio button.
  • If only one role is available, you will still need to select it.
  • Select 'Continue' to move to next step.

Approval route 1: Local administrator approves the request

The National Data Submission Portal (NDSP) uses a partially devolved permissions model. This means some access is managed centrally by NHS England, while other access can be delegated to local administrators within provider organisations. NDSP also uses a many-to-many permissions structure.

A single user account can hold multiple permissions, and each permission is defined by a combination of:

  • Organisation (ODS code)
  • Collection
  • Role

A user may therefore hold different roles for different organisation–collection pairings.

Roles available in NDSP:

  1. Local administrator (approver)

    Local administrators allow devolved, local control of user management so organisations can manage their own submitters.

    • Can grant and remove permissions for submitters and only view users for their specific organisation–collection pairing.
    • Assigned centrally by an NHS England system administrator.
    • Appointment must be approved first by the organisation’s Information Governance Lead, Caldicott Guardian, or Senior Information Risk Owner (SIRO) (as named in the ODS portal).
    • Local admins cannot exceed the scope of the organisation and collection they were assigned.

  2. Submitter:

    This is the standard operational role for users responsible for submitting data.

    • Can submit files for the organisation–collection pairing.
    • Can view their organisational submission history and validation feedback.
    • Can be granted permission by:
      • A local administrator, or
      • NHS England (if the organisation has no local admin in place).
  3. View-only:

    Typical use cases include IG leads, managers, or analysts who need oversight without contributing with submissions.

    • Can view submission history and validation results
    • Cannot submit files.
    • Can be granted by:
      • A local administrator, or
      • NHS England (if the organisation has no local admin).

    4. Granting permissions

    Permissions follow these rules:

    • Each permission is tied to both a collection and an organisation.
    • A user may have different roles for different collections or organisations.
    • A local admin can only manage permissions within the same organisation–collection pairing they hold admin rights for.
    • NHS England administrators can override or assign any permission where needed.

    Note on role availability

    Not all roles in NDSP are available for every collection. Even if a role exists within the platform, it may not be enabled for a particular collection. Role availability is determined by regulatory requirements and governance rules that apply to each individual data collection.
